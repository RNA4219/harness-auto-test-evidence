# AC-20260703-postpoc-007 Live Connector Runtime Evidence

## Scope

Acceptance for `TASK-POSTPOC-007` /
`POSTPOC-PKT-007-live-connectors`.

Implemented a deterministic connector runtime evaluator and safe execution
manifest builder that models dry-run, live, replay, and rollback-preview modes
with fake endpoint contracts, idempotency records, rollback visibility records,
token references, redaction evidence, sourceRef-backed attempts, approval
gates, side-effect steps, and rollback-preview steps. It does not write to real
external systems.

## Requirements

- `HATE-POSTPOC-GAP-007`
- `POSTPOC-PKT-007-live-connectors`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-007-live-connector-runtime`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Fake endpoint live-mode behavior is represented without touching real
  external systems.
- Dry-run outputs are distinguishable from live outputs and perform no side
  effects.
- Idempotent retry preserves the same idempotency key and payload hash.
- Raw token exposure is denied; only token references are allowed.
- Missing idempotency key, unavailable endpoint, and missing rollback
  visibility produce hold findings.
- Rejected connector payloads remain visible and do not become silent pass.
- Dry-run execution plans skip side-effect steps.
- Live execution plans require explicit approval and rollback visibility before
  the provider handoff step can be ready.
- Connector execution manifests can be written as deterministic local
  artifacts.
