# AC-20260703-postpoc-002 Dashboard Interaction Evidence

## Scope

Acceptance for `TASK-POSTPOC-002` /
`POSTPOC-PKT-002-interactive-dashboard`.

Implemented a deterministic dashboard interaction evaluator and read-only
static HTML evidence renderer that model authenticated session views,
dashboard route states, RBAC denial states, unsafe-hidden states, stale
read-model states, and auditable dashboard action intents. It does not
implement a full browser SPA or real auth provider.

## Requirements

- `HATE-POSTPOC-GAP-002`
- `POSTPOC-PKT-002-interactive-dashboard`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-002-interactive-dashboard-frontend`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Portfolio route state is backed by a canonical read model reference.
- Permission denied route rendering is explicitly tied to RBAC denial.
- Stale read-model state is visible.
- Unsafe artifact body and raw connector payload are hidden.
- Manual review action creates an auditable action intent with actor,
  target, idempotency key, and sourceRefs.
- Dashboard cannot recompute verdicts outside canonical read model records.
- Dashboard actions cannot silently mutate local state without an intent.
- Static HTML dashboard evidence can be written as a local artifact.
- Static HTML rendering does not leak unsafe artifact bodies or raw connector
  payloads.
