# AC-20260703-postpoc-013 Capacity Benchmark Evidence

## Scope

Acceptance for `TASK-POSTPOC-013` /
`POSTPOC-PKT-013-capacity-benchmark`.

Implemented a deterministic capacity benchmark evidence evaluator and
regression packet builder that records measured 100 repo, 1000 repo, 1M
findings, cold cache, warm cache, and large monorepo scenarios. The evaluator
requires reproducible dataset hashes, machine profile data, runtime and memory
budgets, timeout counts, explicit degradation modes, and previous-vs-current
regression comparisons.

## Requirements

- `HATE-POSTPOC-GAP-013`
- `POSTPOC-PKT-013-capacity-benchmark`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-013-capacity-benchmark-with-measured-baselines`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Measured baselines include machine profile and reproducible dataset hash.
- 100 repo, 1000 repo, and 1M findings scenarios produce benchmark records.
- Cold cache and warm cache scenarios expose cache-hit and duration behavior.
- Runtime and memory budget exceedance hold the report.
- Timeout or budget failure cannot pass without an explicit degradation mode.
- Non-reproducible dataset references hold the report.
- Capacity regression packets compare current and previous baselines by
  scenario.
- Runtime, memory, cache-hit, missing previous baseline, and budget-status
  regressions block capacity promotion.
- Capacity regression packets can be written as deterministic local artifacts.
