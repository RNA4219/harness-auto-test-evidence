# AC-20260703-postpoc-001 Hosted Scheduler Runtime Evidence

## Scope

Acceptance for `TASK-POSTPOC-001` /
`POSTPOC-PKT-001-hosted-scheduler-runtime`.

Implemented a local hosted scheduler runtime evaluator and dispatch-manifest
builder that reduces scheduler events into worker state, lease events, job
results, dispatch shard entries, retry/resume checks, and hold findings. This
is not a hosted SaaS queue implementation; it is the canonical
evidence/runtime contract needed before a hosted worker can be wired.

- `HATE-POSTPOC-GAP-001`

- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Lease acquisition and heartbeat can complete a job with sourceRefs.
- Stale leases produce hold findings and never become pass.
- Retry recovery preserves resume tokens for partial evidence.
- Cancellation without actor/reason produces hold and writes an event.
- Retry budget exhaustion produces hold.
- Partial result retry without resume token produces hold.
- Failed jobs require failure kind, excerpt, and cleanup evidence.
- Dispatch plans shard jobs deterministically by queue and max concurrency.
- Dispatch entries with partial results require resume tokens.
- Duplicate jobs, exhausted retry budget, invalid concurrency, and empty job
  lists produce hold findings.
- Dispatch manifests can be written as deterministic local artifacts.
