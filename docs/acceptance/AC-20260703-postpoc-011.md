# AC-20260703-postpoc-011 Hosted Multi-Tenant API Evidence

## Scope

Acceptance for `TASK-POSTPOC-011` /
`POSTPOC-PKT-011-hosted-api`.

Implemented a deterministic hosted API evaluator and route-contract artifact
builder that emits request, authz decision, rate-limit records, a platform
route manifest, and an OpenAPI 3.1 style contract for OIDC, API token, and
service-account requests. It models tenant boundaries, token expiry, service
account scopes, rate-limit denial, and non-leaky cross-tenant denial without
running a server.

## Requirements

- `HATE-POSTPOC-GAP-011`
- `POSTPOC-PKT-011-hosted-api`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-011-hosted-multi-tenant-api`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Allowed tenant-scoped requests produce request, authz, rate-limit, and audit
  evidence.
- Cross-tenant access denial is deterministic and does not leak restricted
  tenant data.
- Expired tokens are denied.
- Service account scope violations are denied.
- Rate limit exceedance is explicit.
- Every hosted request requires audit evidence.
- Raw tokens or bad token refs are treated as restricted data leaks.
- Platform API routes for run, history, findings, debt, review, policy, and
  report surfaces are represented as tenant-scoped audited route contracts.
- OpenAPI artifact generation preserves HATE-specific tenant, audit, rate
  limit, resource, action, and scope metadata.
