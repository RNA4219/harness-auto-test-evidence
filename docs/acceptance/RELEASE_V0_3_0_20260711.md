## Distribution policy

- GitHub Release assets are the only supported prebuilt package channel.

## Open risks

- Legacy post-P1a Python modules remain at their historical import paths for source compatibility; public CLI handlers can reach them only through hate.compat.v0_2.
- compat-v0.2 is frozen but still reproduces advisory logic that canonical owner repos may supersede before v1.
- Handoff is a local file contract. HATE does not start an external owner, use the network, or provide orchestration.
- product-ops-evidence HATE/v1 alias domain selection is deterministic but heuristic; the full original payload and source references are retained.
- Owner-side consumer schemas are intentionally minimal and do not copy each owner repo's canonical decision logic.
- Pre-existing dirty worktrees in workflow-cookbook, manual-bb-test-harness, and shipyard-cp, plus concurrent unrelated QEG changes, prevent a global clean-worktree claim.
- PyPI is deliberately not a distribution channel; the GitHub Release assets are the canonical package source.
