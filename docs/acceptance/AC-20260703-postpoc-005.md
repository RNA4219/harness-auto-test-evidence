# AC-20260703-postpoc-005 Real-Repo Roster Maintenance Evidence

## Scope

Acceptance for `TASK-POSTPOC-005` /
`POSTPOC-PKT-005-roster-operations`.

Implemented a local roster maintenance evaluator and execution-manifest
builder that turns real-repo roster metadata into maintenance plans,
environment recipes, quarantine events, scheduler-facing execution entries,
bootstrap requirements, external-hold boundaries, and hold findings. This does
not modify external repositories or perform dependency installation; it creates
the evidence/control layer needed before automated large-roster operation.

## Requirements

- `HATE-POSTPOC-GAP-005`
- `POSTPOC-PKT-005-roster-operations`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-005-real-repo-roster-operations`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Owned repo bootstrap with recipe, retry group, and record floor passes.
- External repo repair requests are denied unless explicitly allowed.
- Stale repos produce quarantine events.
- Dependency bootstrap failures produce quarantine events.
- Missing recipe, retry isolation group, and expected record floor produce hold.
- 100 repo dry-run produces deterministic maintenance plans.
- Clean roster entries become scheduler-ready execution entries.
- Quarantined or external-held repos do not silently become runnable jobs.
- Bootstrap-required entries require explicit bootstrap commands.
- Roster execution manifests can be written as deterministic local artifacts.
