# AC-20260703-postpoc-008 Long-Term History Analytics Evidence

## Scope

Acceptance for `TASK-POSTPOC-008` /
`POSTPOC-PKT-008-history-analytics`.

Implemented a deterministic history analytics evaluator and incremental
materialization planner that turns multi-run history samples into
`history-analytics-query`, `history-analytics-result`,
`history-trend-window`, `history-materialization-plan`, and
`history-materialization-manifest` records. It computes flake rate, evidence
freshness, debt age, repo health, baseline drift, regression cluster count,
and manual review latency without requiring a data warehouse backend.

## Requirements

- `HATE-POSTPOC-GAP-008`
- `POSTPOC-PKT-008-history-analytics`
- `POST_POC_PRODUCTIZATION_DETAIL_SPEC.md#hate-postpoc-gap-008-long-term-history-analytics`
- `WORKFLOW_COOKBOOK_OPERATION_CONTRACT.md`

## Acceptance Criteria

- Trend queries are stable across deterministic input history.
- Flake rate, evidence freshness, debt age, repo health, baseline drift,
  regression cluster count, and manual review latency are materialized.
- Query budget failures are explicit and not hidden as empty pass results.
- Stale data produces a hold finding.
- Too-small history windows produce a hold finding.
- Missing metric sources produce a hold finding.
- Unexplained regression clusters produce a hold finding.
- Incremental materialization plans use stable sample fingerprints and cache
  keys to separate `reuse`, `recompute`, and `drop` actions.
- Materialization manifests can be written as deterministic local artifacts.
